# セキュリティポリシー

## 🛡️ データプライバシー

### ローカル処理
このアプリケーションはすべての処理をクライアント側（ブラウザまたはローカルPC）で実行します：

- ✅ アップロードされた画像はサーバーに保存されません
- ✅ 生成されたWordファイルはメモリ上で処理され、即座にダウンロードされます
- ✅ セッション終了後、すべてのデータは自動的に削除されます
- ✅ ユーザーデータの収集や追跡は一切行いません

### Streamlit Cloudでのデプロイ時
- アップロードされたファイルは一時メモリに保存され、セッション終了時に削除されます
- Streamlitのセッション管理により、ユーザー間でデータが共有されることはありません
- HTTPS接続により通信は暗号化されます

## 🔒 セキュアな使用方法

### 推奨事項
1. **信頼できるソースからのファイルのみを使用**
   - 未知の送信元からのファイルは注意して扱ってください
   
2. **依存関係を最新に保つ**
   ```bash
   pip install --upgrade -r requirements.txt
   ```

3. **個人情報を含むファイル名に注意**
   - ファイル名は部品名として文書に挿入されます
   - 機密情報をファイル名に含めないでください

4. **本番環境でのデプロイ**
   - Streamlit Cloudで公開する場合は、認証機能の追加を検討してください
   - プライベートリポジトリの使用を推奨します

### 制限事項
- 大きなファイル（100MB以上）のアップロードは避けてください
- 悪意のあるスクリプトを含む画像ファイルは使用しないでください

## 🐛 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、以下の方法で報告してください：

### 報告方法
1. **GitHubのIssuesは使用しないでください**（公開されるため）
2. **推奨：GitHubのSecurity Advisoryを使用**
   - https://github.com/yusuke021/photo-to-word-system/security/advisories/new
   
3. **または、リポジトリのメンテナーに直接連絡**

### 報告に含めるべき情報
- 脆弱性の詳細な説明
- 再現手順
- 影響を受けるバージョン
- 可能であれば、修正案

### 対応プロセス
1. 報告を受け取り次第、24時間以内に確認します
2. 脆弱性を検証し、重要度を評価します
3. 修正パッチを開発し、テストします
4. セキュリティアップデートをリリースします
5. 公開前に報告者に通知します

## 🔐 依存関係のセキュリティ

### 使用ライブラリ
- `python-docx` - Microsoft Word文書の生成
- `Pillow` - 画像処理
- `pillow-heif` - HEIC形式のサポート
- `streamlit` - Webアプリフレームワーク

### セキュリティチェック
定期的に以下のコマンドで脆弱性をチェックしてください：

```bash
# pipの脆弱性チェック
pip install safety
safety check

# 依存関係の更新確認
pip list --outdated
```
