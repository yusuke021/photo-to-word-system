# テンプレートファイルの設置方法

## 📋 template.docx について

新規Wordファイルを作成する際に、自動的にテンプレートのフォーマットを適用することができます。

## 🛠️ 設置手順

1. お手元のテンプレートWordファイル（フォーマット設定済み）を準備
2. ファイル名を `template.docx` に変更
3. このプロジェクトのルートディレクトリ（app.pyと同じ場所）に配置
4. Gitにコミット＆プッシュ
   ```bash
   git add template.docx
   git commit -m "Add Word document template"
   git push origin main
   ```

## ✨ 使用方法

### Webアプリの場合
- 「既存のWordファイル」をアップロードしない場合、自動的に `template.docx` が読み込まれます
- テンプレートに含まれる：
  - ヘッダー・フッター
  - ページ設定（余白、用紙サイズなど）
  - スタイル設定
  - 既存のコンテンツ（あれば、その後に表が追加されます）

### デスクトップアプリの場合
- 現在は未対応（今後のアップデートで対応予定）

## 📝 注意事項

- テンプレートファイルが見つからない場合は、空白のWordファイルが作成されます
- テンプレートファイルはGitリポジトリに含まれるため、機密情報は含めないでください
- テンプレート内の既存コンテンツの後に新しい表が追加されます

## 🔄 テンプレートの変更

テンプレートを変更したい場合：
1. `template.docx` を削除または置き換え
2. 変更をGitにコミット
3. 再デプロイ（Streamlit Cloudの場合は自動）

## 例：テンプレートに含める内容

- 会社ロゴ
- 文書タイトル
- 作成日、作成者などのヘッダー情報
- フッターにページ番号
- デフォルトのフォント設定
- 余白やページサイズの設定
